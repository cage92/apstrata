Rich Cloud & Active Data Hub

You can use apstrata to authenticate, orchestrate, integrate and process rules all without any need the traditional application server stack.
Rich Client Development

We coined Client2Cloud as a parallel to the older Client/Server paradigm. In those environments, the application was implemented the entire application in the 'thick' client platform and used the back end for persistence.

We consider that environments promoting true rich client development are only those that allow the application programmer to work in the client side technology to implement the application page flow and business logic.

In contrast, traditional application server stack supported development models require the application developer to work most of the time on the server side to implement client side functionality. This is even true if the front-end is implemented in an Rich Internet Application (RIA) platform such as Flex.
